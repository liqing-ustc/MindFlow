## 详评

### 1. Veo-Act: How Far Can Frontier Video Models Advance Generalizable Robot Manipulation?
- **Authors**: Zhongru Zhang, Chenghan Yang, Qingzhou Lu, Yanjiang Guo, Jianke Zhang, Yucheng Hu, Jianyu Chen
- **Link**: [arXiv](http://arxiv.org/abs/2604.04502v1)
- **Source**: 📄 arXiv
- **Method**: 先用 zero-shot 方案验证 Veo-3 作为 robot planner 的潜力：给定当前观察，Veo-3 预测未来图像序列，再用 inverse dynamics model (IDM) 恢复动作。IDM 仅用随机探索数据训练，无需人类示教。发现 Veo-3+IDM 能生成大致正确的 task-level 轨迹，但低层控制精度不够。进而提出分层框架 Veo-Act：Veo-3 做高层 motion planner，VLA policy 做低层执行器，显著提升 instruction-following 性能。在仿真和真实灵巧手上做了验证。
- **锐评**: 这篇的核心 insight 非常有价值——**前沿视频生成模型已经具备足够的物理理解来指导机器人规划**，但精度还不够做低层控制。分层设计（video model plan + VLA execute）是一个优雅的解决方案，也暗示了未来 world model 和 VLA 的协作范式。关键问题是：Veo-3 是闭源模型，可复现性存疑；IDM 只用 random play 训练的假设在复杂任务上是否 hold 需要更多验证。但方向绝对值得关注。

### 2. E-VLA: Event-Augmented Vision-Language-Action Model for Dark and Blurred Scenes
- **Authors**: Jiajun Zhai, Hao Shi, Shangwei Guo, Kailun Yang, Kaiwei Wang
- **Link**: [arXiv](http://arxiv.org/abs/2604.04834v1)
- **Source**: 📄 arXiv
- **Method**: 提出 E-VLA 框架，将 event camera 流直接融入 VLA 模型，而非传统的从 event 重建图像再处理。构建了开源遥操作平台（DAVIS346 event camera）和真实世界 RGB-event-action 同步数据集。提出轻量级、与预训练兼容的 event 融合策略，研究了 event windowing 和 fusion 方式。实验在极低光照（20 lux）下 Pick-Place 成功率从 0%（纯图像）提升到 60%（overlay fusion）和 90%（event adapter）。
- **锐评**: 这是 **event camera + VLA 的首次系统性验证**，填补了一个重要空白。当前 VLA 在光照/模糊等感知退化场景下完全崩溃，这在真实部署中是致命问题。最惊人的发现是：**哪怕只是简单地将 event map 叠加到 RGB 图上（parameter-free），都能带来巨大改善**，说明 event 流提供的运动和结构信息对 VLA 极其互补。开源平台和数据集是加分项。局限是实验场景偏简单（Pick-Place、Sorting），复杂长horizon任务待验证。

### 3. LIBERO-Para: A Diagnostic Benchmark and Metrics for Paraphrase Robustness in VLA Models
- **Authors**: Chanyoung Kim, Minwoo Kim, Minseok Kang, Hyunwoo Kim, Dahuin Jung
- **Link**: [arXiv](https://arxiv.org/abs/2603.28301)
- **Source**: 📰 HF Daily ⬆️ 63
- **Method**: 构建 LIBERO-Para benchmark，独立变化 action expressions 和 object references 来精细分析 VLA 的语言泛化能力。测试了 7 种 VLA 配置（0.6B-7.5B），发现同义改写导致 22-52pp 的性能下降。关键发现：退化主要由 object-level 词汇变化驱动，简单同义词替换就造成大幅下降；80-96% 的失败是 planning-level 轨迹偏离而非执行错误。提出 PRIDE 指标量化 paraphrase 难度。
- **锐评**: 这篇论文暴露了 VLA 的一个**根本性缺陷**：它们根本没学会语义理解，只是在做 surface-level pattern matching。同义词替换就崩，说明 VLM 预训练带来的语义理解在 fine-tune 后被"遗忘"了——可能是小数据 fine-tuning 导致的过拟合。80-96% 失败在 planning level 这个发现特别重要，说明问题出在 task identification 而非运动控制。**这对所有做 VLA 的人都是警钟**。PRIDE 指标的设计也很实用。

### 4. Adaptive Action Chunking at Inference-time for Vision-Language-Action Models
- **Authors**: Yuanchang Liang, Xiaobo Wang, Kai Wang, Shuo Wang, Xiaojiang Peng, Haoyu Chen, David Kim Huat Chua, Prahlad Vadakkepat
- **Link**: [arXiv](http://arxiv.org/abs/2604.04161v1)
- **Source**: 📄 arXiv
- **Method**: 提出 Adaptive Action Chunking (AAC) 策略，利用 action entropy 作为信号自适应决定 chunk size。高 entropy 时缩短 chunk（增强响应性），低 entropy 时延长 chunk（减少 mode-jumping）。在仿真和真实机器人操作任务上验证，超越 SOTA 替代方案。
- **锐评**: Action chunking 的 chunk size 选择一直是 VLA 部署中的 **实际痛点**——固定 chunk 要么太保守（反应慢）要么太激进（动作不连续）。用 entropy 做自适应判断是一个简洁且合理的方案，符合直觉：不确定时多看一步，确定时一口气执行。关键问题是 entropy 的计算开销和校准问题——不同任务/模型的 entropy 分布差异很大，阈值如何设定？但方法本身 zero-cost 兼容任何 VLA 模型，实用性很强。

### 5. OpenWorldLib: A Unified Codebase and Definition of Advanced World Models
- **Authors**: DataFlow Team, Bohan Zeng, Daili Hua, et al.
- **Link**: [arXiv](https://arxiv.org/abs/2604.04707)
- **Source**: 📰 HF Daily ⬆️ 145
- **Method**: 提出 World Model 的统一定义：以感知为核心、具备交互和长期记忆能力的模型/框架，用于理解和预测复杂世界。系统分类 world model 的核心能力，构建统一推理框架 OpenWorldLib，集成不同任务的模型，支持高效复用和协同推理。
- **锐评**: 这是一个 **定义性工作**而非技术突破。World Model 领域确实需要统一定义——目前 video prediction model、simulator、embodied world model 各说各话。他们提出的定义（perception + interaction + long-term memory）比较合理，但偏宽泛。作为统一推理框架更像工程贡献。值得一看他们对 world model 能力的分类和未来方向分析，但不要期待技术深度。

### 6. VLA-Forget: Vision-Language-Action Unlearning for Embodied Foundation Models
- **Authors**: Ravi Ranjan, Agoritsa Polyzou
- **Link**: [arXiv](http://arxiv.org/abs/2604.03956v1)
- **Source**: 📄 arXiv
- **Method**: 针对 OpenVLA 式 VLA 模型提出 VLA-Forget 混合 unlearning 框架，结合 ratio-aware selective editing（处理感知和跨模态特异性）和 layer-selective reasoning/action unlearning（保留 utility 的同时遗忘目标行为）。联合优化三个目标：targeted forgetting、perceptual preservation、reasoning retention。
- **锐评**: VLA unlearning 是一个**新兴且重要的安全方向**。VLA 部署到真实机器人后，移除不安全/错误行为而不破坏整体能力是现实需求。论文指出 VLA 中知识分布在 vision encoder、projector 和 language backbone 多个模块，单一模块 unlearning 不够——这个 observation 本身有价值。但实验似乎主要在 synthetic 设定下做的，真实场景验证不足。方向值得跟踪。

### 7. SKILL0: In-Context Agentic Reinforcement Learning for Skill Internalization
- **Authors**: Zhengxi Lu, Zhiyuan Yao, Jinyang Wu, et al.
- **Link**: [arXiv](https://arxiv.org/abs/2604.02268)
- **Source**: 🔥 HF Trending ⬆️ 86
- **Method**: 提出将 agent skills（推理时动态加载的程序性知识）内化到模型参数中的 RL 框架。训练时从完整 skill context 开始，通过 Dynamic Curriculum 逐步撤除，最终实现 zero-shot 执行。技能按类别分组，用视觉化 context 呈现。在 ALFWorld (+9.7%) 和 Search-QA (+6.6%) 上优于标准 RL baseline，且每步 context 低于 0.5k tokens。
- **锐评**: 核心问题问得好：**agent 能不能把 skill 从"查手册"变成"本能"？** 渐进式撤除 skill context 的训练策略类似于教学中的 scaffolding，直觉上合理。但 9.7% 的提升幅度不算大，而且 ALFWorld 这类环境相对简单。更重要的是，内化后的 skill 是否 robust、是否会跟其他 skill 冲突，论文似乎没深入讨论。

### 8. A Simple Baseline for Streaming Video Understanding
- **Authors**: Yujiao Shen, Shulin Tian, Jingkang Yang, Ziwei Liu
- **Link**: [arXiv](https://arxiv.org/abs/2604.02317)
- **Source**: 🔥 HF Trending ⬆️ 63
- **Method**: 提出 SimpleStream：仅将最近 N 帧送入 off-the-shelf VLM 的滑动窗口 baseline。在 OVO-Bench 和 StreamingBench 上与 13 个 baseline 对比，仅用 4 帧即达 67.7% 和 80.59% 准确率，匹配或超越复杂 streaming 模型。揭示 perception-memory trade-off：更多历史 context 可提升 recall 但削弱实时感知。
- **锐评**: **典型的"皇帝的新衣"工作**。复杂的 memory/retrieval/compression 模块被一个 4 帧滑动窗口打平，说明现有 streaming video benchmark 可能主要测的是近帧感知而非真正的长程记忆。论文最重要的贡献是提出 benchmark 应该分离 recent perception 和 long-range memory——这对 Video LLM 社区是有建设性的批评。

### 9. ROSClaw: A Hierarchical Semantic-Physical Framework for Heterogeneous Multi-Agent Collaboration
- **Authors**: Rongfeng Zhao, Xuanhao Zhang, Zhaochen Guo, et al.
- **Link**: [arXiv](http://arxiv.org/abs/2604.04664v1)
- **Source**: 📄 arXiv
- **Method**: 提出 ROSClaw 框架，用统一 VLM controller 集成 policy learning 和 task execution。利用 e-URDF 表示异构机器人的物理约束，构建 sim-to-real 拓扑映射。支持数据收集、状态积累、迭代 policy 优化的闭环流程。
- **锐评**: 框架性工作，试图解决异构机器人协作的 semantic-physical gap。e-URDF 表示和 sim-to-real 拓扑映射是有意思的设计点，但整体偏系统集成。缺乏与 strong baseline 的对比，实际场景验证有限。作为工程参考值得一看。

### 10. OpenClaw-RL: Train Any Agent Simply by Talking
- **Authors**: Yinjie Wang, Xuyang Chen, Xiaolong Jin, Mengdi Wang, Ling Yang
- **Link**: [arXiv](https://arxiv.org/abs/2603.10165)
- **Source**: 🔥 HF Trending ⬆️ 149
- **Method**: 利用 agent 交互中的 next-state signals（用户回复、工具输出、GUI 状态变化）作为在线学习信号。提取 evaluative signals（通过 PRM judge）和 directive signals（通过 Hindsight-Guided On-Policy Distillation），实现异步 serve-judge-train 流程。
- **锐评**: 思路有趣——从交互中自动挖掘训练信号。但 149 upvotes 的热度和实际技术贡献不太匹配，更像工程框架。核心假设"next-state signal 足以提供有效 reward"在复杂任务中未必成立。

### 11. Agent Lightning: Train ANY AI Agents with Reinforcement Learning
- **Authors**: Xufang Luo, Yuge Zhang, Zhiyuan He, et al.
- **Link**: [arXiv](https://arxiv.org/abs/2508.03680)
- **Source**: 🔥 HF Trending ⬆️ 140
- **Method**: 提出 Training-Agent Disaggregation 架构，将 agent 执行和 RL 训练完全解耦。定义统一数据接口，提出分层 RL 算法 LightningRL，含 credit assignment 模块，支持多 agent 场景和动态 workflow。
- **锐评**: 又一个 Agent RL 框架。解耦设计和 zero code modification 的卖点更像工程贡献。实验在 text-to-SQL、RAG、math tool-use 上做的，跟 embodied AI 关系不大。

### 12. ClawArena: Benchmarking AI Agents in Evolving Information Environments
- **Authors**: Haonian Ji, Kaiwen Xiong, Siwei Han, et al.
- **Link**: [arXiv](https://arxiv.org/abs/2604.04202)
- **Source**: 📰 HF Daily ⬆️ 23
- **Method**: 构建动态信息环境下的 AI agent benchmark，含 64 scenarios/8 domains/1879 evaluation rounds/365 dynamic updates。测试 multi-source conflict reasoning、dynamic belief revision、implicit personalization 三个维度。
- **锐评**: Benchmark 设计有想法（evolving environment + contradictory sources），但偏 NLP/knowledge agent，跟 embodied AI 关系不大。
